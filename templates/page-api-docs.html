API Documentation

Happy Turtle Processing B2B API Reference

Getting Started

Base URL

https://happyturtleprocessing.com/wp-json/happyturtle/v1

Authentication

All API requests require authentication using API Key and API Secret in the request headers:

X-API-Key: your_api_key
X-API-Secret: your_api_secret
⚠️ Important: Store your API credentials securely. Never expose them in client-side code or public repositories.

Products API

GET /products

Retrieve all available products

Query Parameters:
  • category (optional) - Filter by category (e.g., "Concentrates", "Vape Cartridges")
Example Request: 
curl -X GET "https://happyturtleprocessing.com/wp-json/happyturtle/v1/products?category=Concentrates" \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret"
Example Response: 
{
  "success": true,
  "data": [
    {
      "id": 1,
      "sku": "HTP-LR-001",
      "product_name": "Blue Dream Live Resin",
      "category": "Concentrates",
      "subcategory": "Live Resin",
      "thc_percentage": "78.50",
      "cbd_percentage": "0.80",
      "unit_type": "gram",
      "unit_price": "35.00",
      "minimum_order": 5,
      "stock_quantity": 250
    }
  ],
  "count": 1
}

GET /products/{id}

Retrieve a single product by ID

Example Request: 
curl -X GET "https://happyturtleprocessing.com/wp-json/happyturtle/v1/products/1" \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret"

GET /categories

Get all product categories

Example Response: 
{
  "success": true,
  "data": [
    "Concentrates",
    "Vape Cartridges",
    "Edibles",
    "Pre-Rolls"
  ],
  "count": 4
}

Orders API

POST /orders

Create a new order

Request Body: 
{
  "items": [
    {
      "product_id": 1,
      "quantity": 10
    },
    {
      "product_id": 4,
      "quantity": 25
    }
  ]
}
Example Request: 
curl -X POST "https://happyturtleprocessing.com/wp-json/happyturtle/v1/orders" \
  -H "X-API-Key: your_api_key" \
  -H "X-API-Secret: your_api_secret" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {"product_id": 1, "quantity": 10}
    ]
  }'
Example Response: 
{
  "success": true,
  "message": "Order created successfully and pending approval",
  "order_id": 42
}

GET /orders

Retrieve your orders

Query Parameters:
  • status (optional) - Filter by status (e.g., "pending_approval", "approved", "completed")
  • limit (optional) - Number of results (default: 50)
  • offset (optional) - Pagination offset (default: 0)

GET /orders/{id}

Retrieve a single order by ID

Error Codes

Code Message Description
401 Unauthorized Invalid or missing API credentials
400 Bad Request Invalid request parameters or missing required fields
404 Not Found Resource not found
500 Server Error Internal server error
